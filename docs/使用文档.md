# iceCoder 使用说明

> 最后更新：2026-06-03

本文档集中说明**安装、启动、CLI、测试、Benchmark 与 Web 内置命令**。README 仅保留能力概览与跑分表，不重复命令。

## 简介

iceCoder 是面向本地仓库的 **AI 编程助手运行时**：工具调用 Harness、**L1/L2 双模监管**、文件记忆、多会话 Web、Checkpoint 与遥测。架构与模块见 [`PROJECT-GUIDE.md`](./PROJECT-GUIDE.md)、[`项目介绍.md`](./项目介绍.md)。

## 前置要求

- **Node.js** >= 18（推荐 **22+**）
- **npm**（仓库脚本以 npm 为准）或 pnpm / yarn

## 快速开始

```bash
git clone <repo-url>
cd iceCoder
npm install
cp data/config.example.json data/config.json
# 编辑 data/config.json，填入 API Key（见下文「配置」）
npm run dev
```

- **`npm run dev`**：同时启动 API（默认 **1024**）、Vite 前端（**1025**，代理到 API）及脚本内可选隧道；本机打开 **http://127.0.0.1:1025/**
- 若只需 API 或前端：`npm run dev:api`、`npm run dev:web`
- 生产数据目录默认为 `~/.iceCoder/`；开发期使用项目内 `./data/`

## 配置

首次使用复制模板并编辑（勿提交密钥）：

```bash
cp data/config.example.json data/config.json
```

`data/config.json` 示例（支持任意 OpenAI 兼容 API）：

```json
{
  "providers": [
    {
      "id": "minimax-m3",
      "apiUrl": "https://api.minimaxi.com/v1",
      "apiKey": "sk-fengkuangxingqi-4-vwo50",
      "modelName": "MiniMax-M3",
      "parameters": {
        "temperature": 0.5
      },
      "isDefault": true,
      "supportsVision": true,
      "maxContextTokens": 1000000
    }
  ]
}
```

环境变量（`ICE_DATA_DIR`、`ICE_SUPERVISOR_MODE` 等）见 [`环境变量.md`](./环境变量.md)（英文镜像：[`environment-variables.md`](./environment-variables.md)）。

## 启动方式

| 场景 | 命令 |
|------|------|
| 开发（API + Web UI） | `npm run dev` |
| 仅 API | `npm run dev:api` |
| 仅 Vite 前端 | `npm run dev:web` |
| 交互式（CLI 子命令，含 Tunnel 等） | `npm run iceCoder` 或 `npm run iceCoder:cli` |
| 仅 Web（CLI 入口） | `npm run iceCoder:web` |
| 列出工具 | `npm run iceCoder:tools` |
| MCP 服务状态 | `npm run iceCoder:mcp` |
| 查看/切换默认模型 | `npm run iceCoder:config` |

打包后也可直接：`npx tsx src/cli/index.ts <子命令>`，子命令与 `npm run iceCoder:*` 一致（`start` / `cli` / `web` / `run` / `tools` / `mcp` / `config` / `help`）。

### 一次性任务（CLI `run`）

在目标仓库目录执行（或在 prompt 中指明路径）：

```bash
npm run iceCoder:run -- "修复失败测试并跑通 npm test" --max-rounds 100
```

常用选项：

| 选项 | 说明 |
|------|------|
| `--max-rounds <n>` | 最大 Harness 轮次（默认 100） |
| `--json` | 以 JSON 输出结果 |

终端模式（`cli` / `start`）内置：`/scan`、`/tools`、`/clear`、`/help`、`/quit`。

### 生产构建与启动

```bash
npm run build          # tsc + vite build + npm pack
npm start              # NODE_ENV=production，需先 build
```

## Web 聊天内置命令（`~` 前缀）

在 Web 输入框发送（非 Shell）：

| 命令 | 说明 |
|------|------|
| `~scan` | 手机扫码，与 PC **同会话**同步 |
| `~telemetry` | Harness / 记忆等遥测摘要 |
| `~supervisor` | L1/L2 监管事件（可与遥测同模式查看） |
| `~memory` | 记忆相关状态 |
| `~open` | 打开关联资源 |

遥测原始行见 `data/*/telemetry.jsonl`；REST 详情见 [`项目介绍.md`](./项目介绍.md)。

## 测试与覆盖率

```bash
npm test                  # Vitest 全量，约 ~1,867 条用例
npm run test:coverage     # V8 覆盖率报告（coverage/）
```

近期快照（以本地 `npm run test:coverage` 为准）：

- 全 `src/` 行覆盖约 **78%**
- 模块参考：Harness **~84%**、Supervisor **~95%**、记忆 **~71%**

Harness 回归样例：`npm run eval:agent`（`scripts/agent-eval.ts`）。

## 数据路径

| 环境 | 数据根 |
|------|--------|
| 开发 | 项目内 `./data/` |
| 生产 / CLI 默认 | `~/.iceCoder/` |

可用 `ICE_DATA_DIR` 覆盖；记忆在 `data/memory-files/`，会话在 `data/sessions/` 等。

## 注意事项

1. **API Key**：`data/config.json` 含敏感信息，勿提交版本库或外传。
2. **`data/` 目录**：运行期记忆、上传、会话 checkpoint、遥测等；打包发布时通常排除用户数据。
3. **首次运行**：自动创建 `data/memory-files/` 等子目录。
4. **`npm run dev`**：完整脚本可能依赖本机 `cloudflared` 路径；无隧道需求时用 `dev:api` + `dev:web` 即可。

# iceCoder 环境变量参考

本文档列出 `src/` 与 `scripts/` 中**实际读取**的全部进程环境变量，以及 Web 前端使用的浏览器 `localStorage` 键。每个变量均说明用途、合法取值、默认值与读取位置。

**维护约定：** 新增或删除 env 读取时，同步更新本文档与 [`environment-variables.md`](./environment-variables.md)。

---

## 1. 如何使用

| 方式 | 说明 |
|------|------|
| Shell 导出 | `export ICE_DATA_DIR=./data` 后启动进程 |
| 项目根 `.env` | 若使用 dotenv 类工具加载；iceCoder 本身不内置 `.env` 解析，需由启动脚本或 shell 注入 |
| CLI 旗标 | 部分项可用 CLI 覆盖，如 `--port` 优先于 `PORT` |

**路径解析优先级（CLI）：**

1. 显式 env（如 `ICE_CONFIG_PATH`）
2. `ICE_DATA_DIR` 指定根目录下的派生路径
3. 当前工作目录存在 `data/config.json` → 使用 `./data`
4. 否则回退 `~/.iceCoder`

**注意：** `src/index.ts`（`npm run dev:api`）对部分路径使用固定默认（如 `data/config.json`），不经过 CLI 的 `resolveDataPaths()`。生产/CLI 路径以 CLI 模块为准。

## 3. 路径与数据目录

### `ICE_DATA_DIR`

- **作用：** CLI 数据根目录；派生 config、sessions、memory、output 等路径。
- **默认值：** cwd 存在 `data/config.json` → `./data`；否则 `~/.iceCoder`。
- **示例：** `ICE_DATA_DIR=./data`
- **读取：** `src/cli/paths.ts`、`src/web/chat-ws.ts`、`src/harness/supervisor/event-timeline.ts`

### `ICE_CONFIG_PATH`

- **作用：** LLM 提供者配置 JSON（`providers[]`）；**不含** MCP。
- **默认值：** `{ICE_DATA_DIR}/config.json` 或 `data/config.json`（`index.ts` 直启时）。
- **示例：** `ICE_CONFIG_PATH=./data/config.json`
- **读取：** `src/cli/paths.ts`、`src/index.ts`、`src/web/routes/config.ts`

### `ICE_SYSTEM_PROMPT_PATH`

- **作用：** 覆盖组装用的 `system-prompt.md`；显式设置时即使内容与内置默认相同也会加载。
- **默认值：** `{ICE_DATA_DIR}/system-prompt.md`
- **示例：** `ICE_SYSTEM_PROMPT_PATH=./data/system-prompt.md`
- **读取：** `src/cli/paths.ts`、`src/prompts/load-chat-prompt.ts`

### `ICE_OUTPUT_DIR`

- **作用：** 通用输出目录（导出、生成物等）。
- **默认值：** `index.ts` 直启为 `output`；CLI 为 `{ICE_DATA_DIR}/output`。
- **示例：** `ICE_OUTPUT_DIR=./output`

### `ICE_SESSIONS_DIR`

- **作用：** 会话消息、checkpoint、`session-notes.md` 存放目录。
- **默认值：** `data/sessions`
- **示例：** `ICE_SESSIONS_DIR=./data/sessions`
- **读取：** `src/index.ts`、`src/web/chat-ws.ts`、`src/web/routes/sessions.ts`

### `ICE_MEMORY_DIR`

- **作用：** 项目级长期记忆文件根目录。
- **默认值：** `data/memory-files`
- **示例：** `ICE_MEMORY_DIR=./data/memory-files`

### `ICE_USER_MEMORY_DIR`

- **作用：** 用户级记忆目录（跨项目偏好等）。
- **默认值：** `data/user-memory`
- **示例：** `ICE_USER_MEMORY_DIR=./data/user-memory`

### `ICE_RUNTIME_DIR`

- **作用：** 运行时遥测 JSONL 根目录（`telemetry.jsonl` 等）。
- **默认值：** 未设置时 `{ICE_SESSIONS_DIR}/../runtime`（通常即 `data/runtime`）。
- **示例：** `ICE_RUNTIME_DIR=./data/runtime`
- **读取：** `src/harness/runtime-telemetry.ts`
- **关联文件：** `data/runtime/telemetry.jsonl`、`data/runtime/supervisor-events.jsonl`（后者路径由 supervisor 配置 `persistPath` 决定，默认同目录）

## 4. 双模监管（Supervisor）

### `config.json` → `supervisorMode`（非 env）

- **作用：** 用户可配置的三档监管模式；Web 顶栏按钮与配置 API 写入此字段。
- **合法取值：** `off` | `adaptive` | `strict`
- **默认值：** `adaptive`（字段缺失时由 `supervisor-config.json` 的 `mode` 兜底）
- **示例：**

```json
{
  "providers": [ ... ],
  "supervisorMode": "adaptive"
}
```

- **API：** `GET /api/config` 返回 `supervisorMode`；`PATCH /api/config/supervisor-mode`  body `{ "supervisorMode": "strict" }`
- **读取：** `src/config/main-config-supervisor-mode.ts` → `loadSupervisorConfig()` 覆盖 supervisor-config 的 `mode`

### `ICE_SUPERVISOR_CONFIG_PATH`

- **作用：** `supervisor-config.json` 绝对/相对路径。
- **默认值：** `{ICE_DATA_DIR}/supervisor-config.json`
- **示例：** `ICE_SUPERVISOR_CONFIG_PATH=./data/supervisor-config.json`
- **读取：** `src/harness/supervisor/supervisor-config.ts`

### `ICE_SUPERVISOR_SHADOW`

- **作用：** 影子模式——评估链路运行但不改写 `supervisorPhase`（对照实验）。
- **合法取值：** `1`/`true` → 开启；`0`/`false` → 关闭；非法值回退配置文件。
- **默认值：** 配置文件 `shadow`（示例 `false`）；`mode=off` 时强制 `false`。
- **示例：** `ICE_SUPERVISOR_SHADOW=1`

## 5. HTTP 服务与 Node

### `PORT`

- **作用：** HTTP/API 监听端口。
- **默认值：**
  - `tsx src/index.ts` / `npm run dev:api`：**1024**
  - CLI `web` / `start` / `chat`：**3784**
- **合法取值：** 有效 TCP 端口整数；CLI `--port` / `-p` 优先。
- **示例：** `PORT=3784`

### `NODE_ENV`

- **作用：** `production` 时 Express 托管 `dist/public` 静态 SPA。
- **默认值：** 未设置（开发行为）。
- **合法取值：** 常规 Node 约定；iceCoder 仅显式判断 `=== 'production'`。
- **示例：** `NODE_ENV=production`（`npm start` 脚本已设置）

## 6. 提示词、评测与 LLM 请求

### `ICE_EVAL_MODE`

- **作用：** 评测模式——移除工具向提示段、跳过记忆提取、注入评测专用 system 段。
- **合法取值：** 仅 **`1`** 视为启用（非 `true`/`0` 语义）。
- **默认值：** 关闭
- **示例：** `ICE_EVAL_MODE=1`
- **读取：** `src/prompts/load-chat-prompt.ts`、`src/harness/harness-memory.ts`

### `ICE_DISABLE_TOOLS`

- **作用：** 不向 LLM 提供 tool schema，并移除 `tool_usage` / `shell_guide` 提示段。
- **合法取值：** 仅 **`1`** 视为启用。
- **默认值：** 关闭
- **示例：** `ICE_DISABLE_TOOLS=1`

### `ICE_CONTEXT_WINDOW`

- **作用：** 覆盖「生效上下文窗口」token 上限，影响压缩触发、冰豆圆环、记忆预算档位等。
- **解析优先级：** 本 env → `data/config.json` 默认 provider 的 `maxContextTokens` → 所有 provider 最大值 → **128000**。
- **合法取值：** 正整数；非法则忽略。
- **示例：** `ICE_CONTEXT_WINDOW=200000`
- **读取：** `src/harness/context-window-tier.ts`、`src/harness/harness-memory.ts`、`src/harness/context-compactor.ts`
- **注意：** `readEffectiveContextWindowTokens()` 读 config 时固定 `data/config.json`，未跟随 `ICE_CONFIG_PATH`。

### `ICE_OPENAI_REQUEST_TIMEOUT_MS`

- **作用：** OpenAI 兼容适配器单次 HTTP 请求超时（毫秒）。
- **优先级：** provider.`requestTimeoutMs` → 本 env → 适配器默认 **120000** ms。
- **合法取值：** 正整数；非法忽略。
- **示例：** `ICE_OPENAI_REQUEST_TIMEOUT_MS=180000`

### `ICE_SLIM_TOOL_DESCRIPTIONS`

- **作用：** 向 LLM 发送 tools 前截断过长 `description`（利于前缀缓存、省 token）。
- **合法取值：** `1` / `true` / `yes`（大小写不敏感）→ 开启。
- **默认值：** 关闭
- **示例：** `ICE_SLIM_TOOL_DESCRIPTIONS=1`

### `ICE_SLIM_TOOL_DESC_MAX_CHARS`

- **作用：** 开启 slim 后单条 description 最大字符数。
- **合法取值：** ≥ **48** 的整数；否则回退 **384**。
- **默认值：** **384**
- **示例：** `ICE_SLIM_TOOL_DESC_MAX_CHARS=256`

## 7. Harness 主循环

### `ICE_HARNESS_MAX_ROUNDS`

- **作用：** 单次 `Harness.run()` 最大对话轮次（防无限循环）。
- **合法取值：** 正整数；非法则回退默认。
- **默认值：** **5000**
- **示例：** `ICE_HARNESS_MAX_ROUNDS=200`
- **读取：** `src/harness/token-budget-config.ts`

### `ICE_TASK_GRAPH`

- **作用：** `isTaskGraphEnabled()` 全局 feature flag。
- **合法取值：** 空 / 未设置 / `0` / `false` → **false**；其他任何非空字符串 → **true**（如 `1`、`true`）。
- **默认值：** **false**
- **说明：** 主路径仍由 `shouldUseTaskGraph(intent)` 按 intent 门控（`edit`/`debug`/`test`/`refactor`）；本 flag 对当前主链路影响有限。

### 已移除的环境变量（勿再使用）

| 变量 | 现状 |
|------|------|
| `ICE_HARNESS_TOKEN_BUDGET` | 已移除；固定 **50000000** 累计 token 上限 |
| `ICE_HARNESS_TIMEOUT_MS` / `ICE_HARNESS_TIMEOUT_HOURS` | 已移除；固定 **86400000** ms（24h）墙钟超时 |

## 8. 上下文压缩

### `ICE_COMPACTION_RATIO`

- **作用：** 硬压缩触发阈值 = `上下文窗口 × 比例`。
- **合法取值：** (0, 1] 浮点数；非法则 **0.88**。
- **默认值：** **0.88**

### `ICE_MICRO_COMPACT_RATIO`

- **作用：** 微压缩（本地 stub 工具结果）触发阈值比例。
- **合法取值：** (0, 1]；非法则 **0.72**。
- **默认值：** **0.72**

### `ICE_COMPACTION_RESERVE_TOKENS`

- **作用：** 硬压缩后为近期消息保留的 token 准备金。
- **合法取值：** 正整数；非法则 **15000**。
- **默认值：** **15000**

**读取：** `src/harness/context-compactor.ts`

## 9. 记忆系统

### `ICE_STANDARD_RECALL_COOLDOWN_SEC`

- **作用：** 相同 manifest + 用户问题在冷却期内跳过标准召回（省 LLM/IO）。
- **合法取值：** ≥ **0** 整数；**0** 表示禁用冷却。
- **默认值：** **300**（5 分钟）
- **示例：** `ICE_STANDARD_RECALL_COOLDOWN_SEC=600`

### `ICE_EXTRACTION_MAX_MESSAGES`

- **作用：** 单次 LLM 记忆提取参与的最大 user/assistant 条数。
- **合法取值：** ≥ **20** 整数；非法则 **80**。
- **默认值：** **80**

### `ICE_MEMORY_DIMENSION_DOC`

- **作用：** 记忆提取 prompt 引用的「记忆维度说明」Markdown 路径（相对 cwd 或绝对路径）。
- **默认值：** `docs/记忆系统调整.md`
- **说明：** 文件超过 **12000** 字符会被截断。

**非 env 热配置：** `data/memory/memory-config.json` — 召回预算、Dream 阈值、相关性门控等（`memory-remote-config.ts` 热加载）。

## 10. 工具输出与读文件上限

### `ICE_MAX_TOOL_OUTPUT_CHARS`

- **作用：** 写入 LLM 上下文的单条 tool 结果字符上限（与各工具元数据上限取 min）。
- **合法取值：** **8000–200000**（钳制）；非法则 **24000**。
- **默认值：** **24000**

### `ICE_READ_FILE_MAX_LINES`

- **作用：** `read_file` 未指定 offset/limit 时最多返回行数。
- **合法取值：** **50–5000**（钳制）
- **默认值：** **420**

### `ICE_READ_FILE_MAX_CHARS`

- **作用：** `read_file` 未指定范围时正文软字符上限（与行数同时生效）。
- **合法取值：** **2000–500000**（钳制）
- **默认值：** **18000**

### `ICE_DOC_PARSE_TEXT_MAX_CHARS`

- **作用：** 文档解析等纯文本路径的单次字符软上限。
- **合法取值：** **2000–200000**（钳制）
- **默认值：** **16000**

**读取：** `src/tools/tool-output-limits.ts`

## 11. 子代理（Sub-Agent）

### `ICE_SUBAGENT_TIMEOUT_MS`

- **作用：** `delegate_to_subagent` 整段隔离循环墙钟超时（毫秒）。
- **合法取值：** 正整数
- **默认值：** **120000**（2 分钟；与 OpenAI 默认单次请求超时对齐）

### `ICE_SUBAGENT_CACHE_MAX_ENTRIES`

- **作用：** 进程级 LRU 缓存条目全局上限（跨 taskKey 淘汰最久未访问）。
- **合法取值：** ≥ **1** 整数；非法则 **100**。
- **默认值：** **100**

**读取：** `src/harness/token-budget-config.ts`、`src/harness/sub-agent-runner.ts`

## 12. MCP

### `ICE_MCP_CONFIG_PATH`

- **作用：** MCP Server 配置 JSON（顶层 `mcpServers`）。
- **默认值：** `<cwd>/.iceCoder/mcp.json`

- **作用：** MCP 子进程 `initialize` 握手超时（首次 `npx` 拉包可能较慢）。
- **合法取值：** ≥ **15000** 整数；非法则 **120000**。

**读取：** `src/cli/paths.ts`、`src/mcp/mcp-client.ts`

## 13. Quick Tunnel / 远程访问

### `TUNNEL_URL`

- **作用：** 固定公网访问 URL；设置后跳过 cloudflared metrics 探测。
- **合法取值：** 完整 HTTPS URL（如 `https://xxx.trycloudflare.com`）
- **默认值：** 无（自动探测）
- **读取：** `src/web/routes/remote.ts`、`src/web/tunnel-ready-watcher.ts`

### `ICE_TUNNEL_WS_NOTIFY`

- **作用：** 隧道就绪后是否通过 WebSocket 推送 URL。
- **合法取值：** **`0`** → 关闭；其他 → 开启
- **默认值：** 开启

### `ICE_TUNNEL_PROBE_MS`

- **作用：** 轮询 cloudflared metrics 的间隔（毫秒）。
- **合法取值：** ≥ **500** 整数
- **默认值：** **2500**

### `ICE_TUNNEL_METRICS_HOST`

- **作用：** cloudflared `--metrics` 绑定主机。
- **默认值：** **127.0.0.1**

### `ICE_TUNNEL_METRICS_PORT`

- **作用：** cloudflared `--metrics` 端口；`npm run dev` 脚本示例使用 **20341**。
- **默认值：** **20241**

### `ICE_TUNNEL_METRICS_QUICKTUNNEL`

- **作用：** 完整 metrics quicktunnel URL；设置后覆盖 host+port 拼接。
- **默认值：** `http://{ICE_TUNNEL_METRICS_HOST}:{ICE_TUNNEL_METRICS_PORT}/quicktunnel`

### `CLOUDFLARED_BIN`

- **作用：** CLI `start` 拉起隧道时使用的 `cloudflared` 可执行文件路径。
- **默认值：** 在 PATH 中自动探测
- **示例：** `CLOUDFLARED_BIN=C:\tools\cloudflared\cloudflared.exe`

## 14. 脚本与评测（`scripts/`）

### `ICE_AGENT_EVAL_MODE`

- **作用：** `npm run eval:agent` 运行模式。
- **合法取值：** `mock`（合成指标）| `real`（读 runtime telemetry 的 `summary` 事件）
- **默认值：** **mock**
- **示例：** `ICE_AGENT_EVAL_MODE=real`
- **说明：** 也可用 CLI `--mode mock|real` 覆盖。

### `ICE_RUNTIME_TELEMETRY`

- **作用：** `real` 模式下读取的运行时遥测 JSONL 路径。
- **默认值：** `data/runtime/telemetry.jsonl`

### `ICE_AGENT_EVAL_HISTORY`

- **作用：** eval 报告追加写入的历史 JSONL 路径。
- **默认值：** `data/eval/agent-eval-history.jsonl`

---

## 15. 终端与系统（辅助）

### `NO_COLOR`

- **作用：** 任意非空值禁用 CLI ANSI 颜色输出。
- **读取：** `src/cli/utils/terminal-ui.ts`

### `COMSPEC` / `SHELL`

- **作用：** `env_info` 工具报告当前 shell 路径（Windows / Unix）。
- **默认值：** 系统环境；通常无需修改。

---

## 16. 浏览器 localStorage（非服务端 env）

### `ICE_PLAN_PANEL`

- **作用：** Web 聊天页任务图/计划面板与冰豆底部详情卡片显隐。
- **合法取值：** 键值为 **`0`** → 隐藏；删除键或其他值 → 显示
- **存储：** 浏览器 `localStorage`（非 Node `process.env`）
- **读取：** `src/public/js/chat-execution-plan.js`、`chat-execution-plan-bridge.js`

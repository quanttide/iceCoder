# 长期记忆提取维度（单一事实来源）

> 由 `memory-llm-extractor` 在提取时读入；编辑本文件即可调整策略，无需改代码大段提示词。
>
> **白名单原则**：长期记忆只承载三类信息——**用户习惯**、**可复用的排错/解决模式**、**项目简介**。任务过程、安装进度、命令流水账留在 **session-notes**，不自动升格。

## 一、用户画像（`type`: user）

| 维度 | 记录内容 | 示例 | 不记 |
|------|----------|------|------|
| 沟通与代码习惯 | 多次出现或用户**明确说**的偏好（沟通、代码、Git、测试等） | commit 用中文、别用分号 | 单次指令；提到 mysql/react 不等于偏好 |
| 技术栈 | 用户**明确声明**的主要语言、框架偏好 | 偏好 TypeScript + Vitest | 本次任务临时选用的工具 |
| 角色与职责 | 职业身份、工作领域（稳定） | 前端架构师 | 本会话临时角色 |
| Git / 测试 / 文档习惯 | 重复出现的流程偏好 | Conventional Commits、先测再改 | 一次性命令 |

## 二、项目简介（`type`: project，`memoryCategory`: project_convention）

| 维度 | 记录内容 | 示例 | 不记 |
|------|----------|------|------|
| 项目目标 | 目的、核心功能、README 未写的用户意图 | 本项目是 AI 助手，单体架构 | 目录复述、package.json 已有命令 |
| 技术架构 | 架构风格、主要模块（稳定约定） | 单体 Runtime、文件持久化 | 安装实例（路径/版本/进度） |
| 依赖与约束 | 关键依赖、禁止的库（用户明确） | 禁止 lodash、Node 18+ | 本次安装的 mysql 8.x 路径 |
| 测试策略 | 仓库级测试约定（非 obvious） | Vitest、npm test 全量 | 本会话跑过的具体命令列表 |

**每项目 1 个** `*-overview.md` 为宜；勿为安装/部署过程创建 sibling 文件。

弱化项（默认**不**因对话出现就提取）：启动命令、外部服务实例、环境安装进度。

## 三、行为反馈（`type`: feedback）

| 维度 | 记录内容 | 示例 | 不记 |
|------|----------|------|------|
| 可复用工作流 | 用户纠正/确认的「当 X，期望 Y」 | Smart Mode 被拦时改用审批卡片 | 本次报错栈、安装到第几步 |
| 纠正过的错误 | 被纠正的技术判断、工具选择 | 「不要用 Jest，用 Vitest」 | 一次性 typos |
| 确认过的正确行为 | 用户明确满意的做法 | 「对，async 错误就这样处理」 | 普通「好的」 |

## memoryCategory 与上述对应

- **user 画像** → `stable_preference` / `habit` / `explicit_rule`；兴趣类 `hobby`（少用）
- **project** → 必须 `project_convention`（与 `type: project` 成对）
- **feedback** → `recurring_mistake` 或 `stable_preference`

**tags**：`dimension:tech_stack`、`dimension:git`、`dimension:feedback_correction` 等。

**confidence**：写盘 ≥ 0.6；**推断**的用户偏好建议 ≥ 0.75，否则归入 feedback 或丢弃。

**与代码门控一致**：`type: project` 必须搭配 `memoryCategory: project_convention`；其它组合丢弃。

## 1. 待优化清单（P0 / P3）

> **P0**：影响可维护性陡变，强烈建议 L3 前修；
> **P3**：可选 / V2 留白。

### P0-1 · `supervisor-bridge.ts` 单文件 godclass

**现象**：`SupervisorRuntimeBridge` 聚合 observe / drift / evaluate / dispatch / boundary / budget / hint / firstRound / checkpoint / restore / reset / mainPath / shadow 等职责；`supervisor-bridge.test.ts` 1500+ 行，改动需 grep 整类。

**影响**：

- 新需求（L3 / V2）易继续涌入单文件；
- 单元测试粒度粗，回归噪声大；
- AI 辅助 context 过大。

### P3-1 · M4 RiskEvaluator（LLM 灰区）未做

**现状**：规格标记 V2 可选；`goalDrift.llmGrayZoneLow/High` 字段已有但无消费者。

### P3-3 · rollback 走 confirm 串联未与 Supervisor 接通

**现状**：[`双模落地缺口.md`](./双模落地缺口.md) §4.4 「❌ 未与 Supervisor 串联」。规格 §19.5 要求 rollback 走 `run_command` 或专用工具 + confirm 权限。

### P3-4 · `EventTimeline` `FileTimelineSink` Promise chain 单调累积

**现状**：`event-timeline.ts:142-152` `this.pending` 一直 chain；长跑 daemon / web server 可能 chain length 增长。

## 3. Benchmark 指标

> 实现后写入 `data/runtime/supervisor-benchmark.jsonl` 或 eval 输出目录。

### 3.1 必须记录

| 指标键 | 含义 |
|--------|------|
| `success_rate` | 任务是否达到用户目标 / 验证通过 |
| `recovery_count` | takeover 后进入恢复主路径（§10）次数 |
| `switch_frequency` | `free` ↔ `takeover` ↔ `handoff` 切换次数（shadow 含「本会切换」） |
| `wasted_token` | 无效探索、重复失败、被 block 仍消耗的 token 估计 |
| `user_retry` | 用户在同 goal 上重发 / 重开会话次数 |
| `rollback_count` | 经 confirm 的工作区回滚次数 |
| `drift_incidents` | `goal_drift` / `scope_creep` 信号触发次数 |

### 3.2 建议附加字段

| 字段 | 说明 |
|------|------|
| `mode` | `ICE_SUPERVISOR_MODE` |
| `shadow` | 是否 shadow 运行 |
| `domain` | `TaskDomain` |
| `takeover_reason` | 首次 takeover 的 `DeviationSignal[]` 摘要 |
| `recovery_rounds_used` | 相对 `maxRecoveryRounds` |
| `handoff_success` | 是否完成交还 |

### 3.3 导出与对照

- **CSV：** 按任务一行，供 spreadsheet / CI trend
- **Shadow 对照：** 同一输入下 `shadow=true` vs `shadow=false` 的 `switch_frequency` 差分
- **版本标签：** git commit 或 package version 写入每条记录

## 4. Learning 层（概要）

**原则：** 离线建议、在线冻结；Learning **不得**在任务中途改写 `supervisorPhase` 或绕过 `ToolGate` / `CorrectionPort`。

```text
EventTimeline + eval history
        ↓
  LearningAnalyzer（批处理 / CLI）
        ↓
  建议：riskThreshold、triggers.*、recovery 预算
        ↓
  人工 review → 合并 supervisor-config.json
```

**可选信号源：**

- shadow 中「本会 takeover 但未接管」→ 阈值是否过松
- `recovery_count` 高但 `success_rate` 低 → 模板图 / 降级路径问题
- `wasted_token` 与 `switch_frequency` 双高 → 稳定窗口 / 冷却参数

**V1 不做的：** 在线 RL、自动 merge 配置、跨租户聚合。

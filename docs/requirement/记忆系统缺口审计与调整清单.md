# 记忆系统调整需求

> **状态**：已实施（2026-06-10 全量修复 + 测试通过）
> **日期**：2026-06-10  
> **关联**：[记忆系统调整-finish](./记忆系统调整-finish.md) · [记忆整合 Dream 方案](./记忆整合Dream优化方案-finish.md)

---

## 1. 背景

长期记忆（`memory-files/`、`user-memory/`）当前**写入过宽**：ops 任务（如 zip 装 MySQL）一次会话可 Extract 多轮、写 7+ 条，多为安装进度/命令流水账，与产品目标不符。

**目标**：长期记忆只承载三类信息——**用户习惯**、**可复用的排错/解决模式**、**项目简介**；任务过程留在 **session-notes**，不自动升格。

---

## 2. 范围定义

### 2.1 长期记忆（白名单）

| 类型 | 记 | 不记 |
|------|----|------|
| **用户习惯** `user` | 多次出现或用户**明确说**的偏好（沟通、代码、Git、测试等） | 单次指令；提到 mysql/react 不等于偏好 |
| **排错模式** `feedback` | 用户纠正/确认的可复用工作流（「当 X，期望 Y」） | 本次报错栈、安装到第几步 |
| **项目简介** `project` | 目标、架构、README 未写的用户意图；**每项目 1 个** `*-overview.md` | 目录复述、package.json 已有命令、安装实例（路径/版本/进度） |

### 2.2 会话记忆（非长期）

`session-notes.md` 负责：Current State、Workflow、Errors & Corrections、Learnings。  
安装进度、临时命令、本会话排错过程**只写这里**，Extract 默认不升格。

## 3. 功能需求

### 3.1 写入与 Extract（P0 — 优先）

| ID | 需求 | 主要改动 |
|----|------|----------|
| **REQ-E1** | 重写提取策略文档与 prompt，与 §2.1 白名单一致；弱化维度表中的「启动命令 / 外部服务实例」 | `记忆系统调整-finish.md`、`EXTRACTION_CORE_PROMPT` |
| **REQ-E2** | **禁止**因出现 mysql/docker/vite 等关键词就 Extract（turn≥1）；仅：信号词、用户 feedback、满足深度门控 | `harness-memory.ts` `shouldExtract()` |
| **REQ-E3** | 接入已配置但未使用的 `minTokens`、`toolCallInterval`；ops 类任务（安装/部署/下载/配置）默认跳过 Extract | `shouldExtract()` + `loopEndTaskIntent` |
| **REQ-E4** | 每会话成功 Extract ≤1 次；单次 `maxMemories` 15 → **3** | `harness-memory.ts`、`memory-config.ts` |
| **REQ-E5** | 写盘拒绝：文件名含 `current-state`/日期；安装进度快照；纯命令列表型 project；非 feedback 且 confidence&lt;0.75 的推断偏好 | `memory-llm-extractor.ts` |
| **REQ-E6** | 主代理未要求 remember 时不得 `write_file` 写 memory；ops 过程只更新 session-notes | `memory-prompt.ts` |
| **REQ-E7** | casual 提取：`allowContentSignalWithoutTools: false` | `memory-config.ts` |

**Extract 触发顺序（目标态）**：默认不写 → 信号词 / feedback → 长对话 + tool 轮 + 节流。**禁止**名词启发式、ops 任务默认 Extract。

### 3.2 索引与可观测（P1）

| ID | 需求 | 主要改动 |
|----|------|----------|
| **REQ-I1** | `MEMORY.md` 缺失时 bootstrap，Extract 写盘必 upsert 索引 | `memory-index-maintainer.ts` |
| **REQ-I2** | 主代理 `write_file`、手动 save、Web 删盘与 Extract 走同一套索引维护 | `file-memory-manager.ts`、Harness、Web DELETE |
| **REQ-I3** | 淘汰/删盘后 `removeIndexRows`，避免索引孤儿 | `memory-eviction.ts`、Web |
| **REQ-I4** | 遥测：`memory_index_rebuild`、`logDream.skipReason` | `memory-telemetry.ts` |

### 3.3 召回与 Dream（P2）

| ID | 需求 | 主要改动 |
|----|------|----------|
| **REQ-R1** | Recall LLM 前关键词粗筛；降低 `LLM_RECALL_MIN_CANDIDATES` 或分层 | `memory-recall.ts` |
| **REQ-R2** | Dream 空跑退避；观察后 `ICE_RULE_MERGE=merge` 合并重复 project 简介 | `memory-dream.ts`、运维 |
| **REQ-R3** | 主代理写记忆路径接入秘密扫描（与 Extract 一致） | Harness / `memory-secret-scanner.ts` |

### 3.4 可选（后续）

- 写盘信息密度二次打分（REQ-E5 增强）
- session `Learnings` 中带「以后/总是/别」的句子升格为长期记忆
- LoCoMo 召回增强（同义词、实体索引、CoN 引导等）

## 4. 约束

| 项 | 要求 |
|----|------|
| Extract 去重 | 已 hardcode merge @ 0.85（无 `ICE_EXTRACT_DEDUP` 环境变量） |
| Dream 规则合并 | 默认 `ICE_RULE_MERGE=shadow`，需运维切换为 `merge` |
| confidence 下限 | 写盘 ≥ 0.6；推断偏好建议 ≥ 0.75 或归入 feedback |
| 维度表 | `type: project` 必须配 `memoryCategory: project_convention` |

---

## 5. 验收标准

### 5.1 场景

| 场景 | session-notes | 长期记忆 |
|------|:-------------:|:--------:|
| zip 装 MySQL，成功 | 有状态/错误/命令 | **0～1 条**（仅可复用 feedback，如工具被拦怎么处理） |
| 「记住，commit 用中文」 | 可选 | **1 条** user → `user-memory` |
| 「本项目是 AI 助手，单体架构」 | 可选 | **1 个** overview 文件，无安装 sibling |

### 5.2 指标

| 指标 | 目标 |
|------|------|
| ops 任务 Extract 写盘 | 0～1 条/会话 |
| MEMORY.md 孤儿 | ≤ 5 |
| Dream `executed=false` 占比 | &lt; 15%（含退避） |

用户要创建、生成、编辑、修改或调整技能时，按以下流程操作。

## 唯一目录

所有技能文件的读写**只能**在 `ICE_SKILLS_DIR` 目录（与用户记忆同级，环境变量 `ICE_SKILLS_DIR`）。

- **禁止**写入 `src/skills/`、`.cursor/skills/`、`docs/`、项目根目录或其他路径
- 编辑已有技能时：先 `read_file` 原文件，再在同一目录内修改，不要复制到别处

## 文件规则

- 一个技能 = 一个 `.md` 文件
- **根目录**：`技能名.md`（如 `创建技能.md` → `#创建技能.md`）
- **一级子文件夹**（可放配套脚本等资源）：`文件夹/技能.md`（如 `openClaude/skll.md` → `#openClaude/skll.md`）；只扫描该文件夹内直接的 `.md`，不递归更深层
- 用 `write_file` / `edit_file` 创建或更新，无需重启服务

## 必填格式

YAML frontmatter + 正文：

```markdown
---
name: 显示名称
description: 一句话说明何时使用此技能
createdAt: 2026-06-16T00:00:00.000Z
---

（给 Agent 的步骤、约束、示例…）
```

## 编写建议

1. **先澄清**：用途、触发场景、输出格式；用户给了原文则尽量原样写入正文
2. **name**：简短中文或英文标识
3. **description**：一句话，便于 `#` 下拉和技能库展示
4. **正文**：分步指令、禁止事项、示例；避免空泛描述
5. **保存后**：告知用户文件名，并说明可在聊天 `#` 或侧栏「技能」中选用

## 自检

- [ ] 文件在 `ICE_SKILLS_DIR` 下
- [ ] 扩展名为 `.md`
- [ ] frontmatter 含 `name`、`description`
- [ ] 正文是可执行的步骤，而非泛泛介绍

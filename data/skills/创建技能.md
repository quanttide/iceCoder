用户要创建、生成、编辑、修改或调整技能时，按以下流程操作。

## 唯一目录

所有技能文件的读写**只能**在 `ICE_SKILLS_DIR` 目录（与用户记忆同级，环境变量 `ICE_SKILLS_DIR`）。

- **禁止**写入 `src/skills/`、`.cursor/skills/`、`docs/`、项目根目录或其他路径
- 编辑已有技能时：先 `read_file` 原文件，再在同一目录内修改，不要复制到别处

## 两种布局

### 1. 纯指引技能（无脚本）

单个 `.md` 放在 `ICE_SKILLS_DIR` 根目录：

```
ICE_SKILLS_DIR/
└── 创建技能.md          → 聊天引用 #创建技能.md
```

### 2. 带脚本的技能（推荐文件夹）

**一个技能 = 一个一级文件夹**，Markdown 固定命名为 `skill.md`，脚本与资源放在同目录：

```
ICE_SKILLS_DIR/
└── connectWeChat/
    ├── skill.md           → 聊天引用 #connectWeChat/skill.md（唯一会被扫描注册的 .md）
    ├── connect.js         → 主脚本（与 skill.md 同级）
    └── scripts/
        └── connect.js     → 或放在 scripts/ 子目录（可选，不单独注册为技能）
```

约定：

- **文件夹名**：英文 camelCase 或语义化标识（如 `connectWeChat`、`openClaude`）
- **技能正文**：必须是 `文件夹/skill.md`，不要用随意文件名（如 `skll.md`）
- **禁止**在技能文件夹内创建 `README.md` / `readme.md`（一级子目录内每个 `.md` 都会被扫描注册为独立技能，会出现多余的 README 条目；说明一律写在 `skill.md` 正文里）
- **脚本路径**：优先 `文件夹/主逻辑.js`；步骤多或需拆分用 `文件夹/scripts/*.js`
- **正文里写清**：脚本相对路径、如何执行（`node connectWeChat/connect.js` 等）、依赖与前置条件
- 只扫描一级子目录内**直接的** `.md`；`scripts/` 下的文件不会被当成独立技能

示例（连接微信）：

| 路径 | 作用 |
|------|------|
| `connectWeChat/skill.md` | 步骤、约束、何时调用脚本 |
| `connectWeChat/connect.js` | 固定流程的可执行脚本 |
| `connectWeChat/scripts/connect.js` | 同上，脚本较多时的备选布局 |

## 必填格式

`skill.md` 或根目录 `.md` 均使用 YAML frontmatter + 正文：

```markdown
---
name: 连接微信
description: 通过脚本连接并操作微信客户端
createdAt: 2026-06-16T00:00:00.000Z
---

1. 确认 `connectWeChat/connect.js` 存在
2. 执行 `node connectWeChat/connect.js` …
```

## 编写建议

1. **先澄清**：用途、触发场景、是否要配套脚本
2. **有脚本就用文件夹 + `skill.md`**；纯文字指引才用根目录单文件
3. **name / description**：便于 `#` 下拉与技能库展示
4. **正文**：可执行步骤；脚本技能须写明脚本路径与运行方式
5. **保存后**：告知引用方式（如 `#connectWeChat/skill.md`）及脚本位置

## 自检

- [ ] 均在 `ICE_SKILLS_DIR` 下
- [ ] 带脚本：`文件夹/skill.md` + 同目录（或 `scripts/`）下的 `.js`
- [ ] 技能文件夹内**没有** `README.md` / `readme.md`
- [ ] 无脚本：根目录 `xxx.md` 即可
- [ ] frontmatter 含 `name`、`description`
- [ ] 正文可执行，脚本路径与命令已写清

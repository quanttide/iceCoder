为用户指定的模块编写单元测试，流程如下：

1. **先读代码**：弄清导出函数、边界条件、依赖与现有测试风格
2. **对齐框架**：沿用项目已有测试库（如 vitest / jest / junit）和目录约定
3. **覆盖重点**：
   - 正常路径
   - 边界值与空输入
   - 错误分支与异常
4. **命名清晰**：`describe` / `it` 用中文或英文均可，但需表意明确
5. **跑通测试**：编写后执行对应 test 命令（如 `npm test` / `mvn test` / `pytest`），失败则修复直至通过

输出时说明：新增了哪些用例、覆盖了什么场景、如何单独运行该测试文件。

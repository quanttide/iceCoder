iceCoder 宠物状态设计方案
1. 设计原则
状态覆盖完整：覆盖 Harness 主循环、记忆系统、子代理、压缩、检查点等所有核心模块的关键节点。

视觉区分清晰：不同系统使用不同的宠物动作/表情基调（记忆系统偏“回忆/思考”，Harness 偏“执行/动作”，压缩偏“整理/收纳”）。

气泡分级：重要信息（记忆召回、任务完成）用长气泡，次要信息（轮次计数、压缩完成）用短气泡或仅表情。

不打扰用户：高频事件（如每轮 LLM 调用）仅用表情变化，不弹气泡。低频高价值事件才弹气泡。

2. 宠物基础状态帧
帧名	视觉描述	触发场景
idle_1	站立，眼睛正常，缓慢眨眼	默认空闲状态
idle_2	站立，眼睛微闭（交替帧）	空闲动画循环
sleep	蹲下，眼睛闭，出现“Z”符号	超过 30 秒无活动
wake	从蹲下站起，揉眼睛	从休眠中唤醒
thinking	眼睛看向右上方，身体微倾	LLM 推理中
working	身体上下微动，手臂抬起，有速度线	工具执行中
happy	跳起，眼睛弯成月牙，双手举起	任务成功 / 记忆命中
confused	歪头，头顶出现“？”	失败 / 错误 / 用户纠正
alert	站直，眼睛睁大，头顶感叹号	需要用户确认
3. Harness 主循环状态
状态	宠物反应	气泡内容（如有）	触发频率
循环启动	alert → thinking	无	每轮
LLM 推理中	thinking（持续）	无（高频，不弹气泡）	每轮
工具执行中	working（持续），可随机变体：working_read（眼镜）、working_write（笔）、working_shell（齿轮）	无	每轮
本轮完成（无错误）	idle_1，微点头	无	每轮
验证门禁触发	站直，手放在下巴上，眼睛眯起	“让我验证一下修改是否正确…”	修改代码后
验证通过	happy（短暂）	“✅ 验证通过！”	验证完成后
验证失败	confused	“验证失败，正在排查…”	验证失败后
重复失败检测	confused → thinking	“这个工具连续失败了，换个思路…”	连续 3 次失败
熔断器触发	坐在地上，头顶冒烟	“连续失败太多次，已暂停执行。请检查环境。”	极少
4. 记忆系统状态
状态	宠物反应	气泡内容	触发频率
记忆召回开始	眼睛变成旋转的加载圈	无	每次召回
记忆召回命中	眼睛发光，身体微微后仰，然后点头	“我想起了你之前说过…（列出 1-2 条记忆关键词）”	每次成功注入
记忆召回为空	快速眨眼，轻微摇头	无（高频，不弹气泡）	频繁
新记忆自动提取	低头做笔记动作，然后微笑	“💾 已记住：你偏好 Vitest”	首次提取
用户纠正记忆	confused → 快速记录动作 → 点头	“已更正，下次注意！”	用户说“不对”时
Dream 整合开始	躺下，闭上眼睛，出现“Z” → 睡醒，伸懒腰	“正在整合记忆库（Dream）…”	后台触发
Dream 整合完成	站起来，身体清爽，头顶小星星	“记忆整理完成，合并了重复条目。”	Dream 结束
记忆淘汰触发	打开一个小抽屉，把东西放进去	“清理了一些不常用的旧记忆。”	超过上限时
5. 子代理状态
状态	宠物反应	气泡内容	触发频率
子代理委派	从口袋掏出一个小分身，挥手告别	“已派助手去探索 {目录/模块}…”	每次调用 delegate
子代理运行中	宠物看表或来回踱步，小分身在远处冒泡	无（高频）	子代理每轮工具
子代理完成	小分身跑回来，递上一份报告，宠物阅读后点头	“探索完成：{文件数} 个文件，摘要已生成。”	每次子代理返回
子代理超时/失败	小分身沮丧地空手回来，宠物拍拍它安慰	“探索超时，已返回部分结果。”	异常时
6. 上下文压缩状态
状态	宠物反应	气泡内容	触发频率
微压缩执行	快速整理桌面，把纸堆推整齐	无（后台，不弹气泡）	65% 阈值
硬压缩执行	拿出一个大箱子，把文件整齐码进去	“上下文已压缩，保留了关键任务状态。”	88% 阈值
压缩后恢复	从箱子里迅速翻出一张任务卡片，回忆状	“任务恢复：{当前目标}，继续执行。”	每次硬压缩后
7. 检查点 / 断点续传状态
状态	宠物反应	气泡内容	触发频率
检查点保存	在笔记本上盖章	“任务状态已存档。”	进程退出前 / 压缩时
断点恢复	翻到笔记本的某一页，仔细阅读，然后站起	“检测到未完成的任务：{目标}，从第 {N} 轮继续。”	新会话启动时
8. 特殊情况状态
状态	宠物反应	气泡内容	触发频率
用户长时间无输入	sleep	无	30s+ 无活动
任务完成（model_done）	happy + 撒花动画	“任务完成！”	每次正常结束
任务超时/预算耗尽	看手表，然后摊手	“任务时间到了，已完成的部分已存档。”	极少
危险操作确认弹窗	alert + 身体前倾，手指向屏幕	“⚠️ 即将执行破坏性操作，请确认。”	用到危险工具时
9. 气泡规则
高频事件：每轮 LLM 调用、工具执行，不弹气泡，仅表情变化。

中频事件：记忆召回、子代理委派，简短气泡，2-3 秒后消失。

低频高价值事件：Dream 整合、硬压缩、验证完成、断点恢复，长气泡，4-5 秒后消失，可叠加历史记录。

需用户操作事件：危险确认、熔断，气泡不自动消失，直到用户操作。

同时只显示一个气泡，优先级：危险确认 > 熔断 > 验证结果 > 记忆提示 > 其他。

10. 实现建议
分期开发：第一期实现基础帧 + Harness 循环 + 记忆召回；第二期加入子代理和压缩状态；第三期补充 Dream、检查点和特殊情况。

技术选型：使用 Canvas 绘制黑白点阵图，宠物尺寸 48×32 或 32×32 像素，风格统一为单色 LCD 电子宠物屏。

性能：高频事件仅触发 Canvas 帧切换，不涉及 DOM 操作；气泡采用绝对定位的 DIV，复用实例。
